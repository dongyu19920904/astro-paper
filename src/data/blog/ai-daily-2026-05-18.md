Codex 最近有个新玩法：让它回看你的历史操作记录，自动把你重复干的事打包成可复用的 Skill。有人试了一下，整理出 4 个 Skill，说其中一个"这他妈就是我最需要的"。我看到这条消息的时候，第一反应不是"哇好厉害"，而是——等等，这玩意儿在观察我？

然后我想起来，我自己就是那个被"观察"的人。

## Table of contents

## 我其实一直在重复同样的事

卖账号这件事，表面上看每天都不一样——今天上新 Kiro，明天补货 Windsurf，后天处理 Claude 断供的售后。但如果有个 Agent 真的回看我过去 130 天的操作记录，它大概会发现：

我每天在重复的事，其实就那么几件。

补货流程：供应商那边有货 → 测号 → 上架 → 改价格 → 发公告。这套流程我做了不知道多少遍，每次都是手动，每次都要想一遍顺序，每次都有可能漏一步。

售后流程：用户说号用不了 → 问他用的什么设备 → 让他截图 → 判断是账号问题还是用户操作问题 → 换号或者教他用。这套我已经背下来了，但还是每次都要打一遍字。

还有内容更新、发卡站维护、日报整理……

如果 Codex 真的能把这些蒸馏成 Skill，我愿意立刻把我的操作记录喂给它。

## gstack 单日 10 万 Star，我的第一反应是进货

[gstack](https://github.com/garrytan/gstack) 这个项目，YC 掌门人 Garry Tan 的 Claude Code 配置，23 个工具分饰 CEO、设计师、工程师等角色，单日 Star 数冲到将近 10 万。

我看到这条的第一反应，说实话，不是"哇这个工作流好厉害"。

是：Claude 账号要不要备货？

这不是玩笑。每次有这种级别的项目爆出来，Claude 的用量就会有一波明显拉升。我之前有过几次没提前备货，结果供应商那边一下子紧张，补货慢了两天，那两天的单子都得延迟发货，售后压力直接翻倍。

所以现在我看 AI 新闻，有一个很本能的反应：这条新闻会不会带动某个账号的需求？

gstack 这个项目，答案是：会。而且是 Claude 账号，不是 ChatGPT。

## "20亿Token捐赠"这个事，我笑了很久

三个00后回母校，搞了个"20亿Token捐赠仪式"，大屏一打，支票一举，校领导站台，场面像捐了半栋教学楼。实际上按粗算大概100块钱。

我笑完之后，想了一下，发现这事其实挺有意思的。

Token 这个单位，现在真的已经变成一种可以被包装的"货币"了。它的实际价值模糊，但数字可以很大，听起来很唬人。这和我卖账号的逻辑有点像——我卖的"无限续杯"，用户其实也不知道自己到底用了多少 Token，他们买的是"不限量"这个感觉，不是具体的数字。

所以那三个00后其实挺懂的。他们卖的不是100块钱，是一个仪式感，是一个"我们这届毕业生懂 AI"的信号。

我当年在闲鱼卖提示词，9.9 一份，写半天，卖的也是一种感觉——"你买了这个，你就懂 AI 了"。后来发现这个感觉不值钱，才转去卖账号。

## 信息差这件事，我每天都在吃

马耳他公民可以免费领一年 GPT，OpenAI 推荐的 YouTube 视频国内打不开——这两条新闻放在一起，有点让人沉默。

我做这行，信息差是我的生意基础之一。国内用户拿不到账号，我帮他们拿；国内用户不知道怎么用，我写教程告诉他们。某种程度上，"墙"是我的护城河。

但我也是那个每天要翻墙才能看完整信息的人。

有时候我会想，如果有一天这个信息差消失了——账号随便买，工具随便用，视频随便看——我的生意还剩什么？

想了一下，大概还剩：帮人省时间、帮人做选择、帮人搞定售后。

这三件事，短期内 AI 还替代不了。

至少现在还不行。
