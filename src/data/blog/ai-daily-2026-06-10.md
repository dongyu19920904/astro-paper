## 反蒸馏这招，其实很聪明

说实话，我第一眼看到这条消息的时候，职业反应比技术反应快多了——

*我手上这批 Claude 账号，有没有影响？*

答案是暂时没有。Fable 5 反蒸馏针对的是拿 API 输出批量训练自己模型的那类人，不是普通用户，不是我这种卖号的，也不是买了号写代码、写文章的用户。真正受影响的是国内那些靠蒸馏闭源模型来"追赶"的团队——也就是把 Claude 输出当训练数据，然后说自己训出了类 Claude 能力的那波操作。

Anthropic 这次埋的机制更损：不封号，不提示，而是悄悄给你输出质量降级。你还以为在正常工作，但其实喂进去的全是差料。

这让我想起了一件事。以前在闲鱼，有个供应商给我供 ChatGPT 号，前两周用着没问题，后来陆续有买家反馈说回复质量变差，总是绕弯子、动不动说"作为 AI 我无法……"。我当时以为是账号被官方限速了，换了一批新的才解决。但现在回想，会不会当时那批号已经触发了某种质量降级？我永远不知道。

这种"让你用，但给你喂差料"的思路，比直接封号狠得多。因为你没有任何明显信号，只有一种说不清楚的"感觉最近效果不太对"。

## 卖铲子的人，不需要太担心这个

但我自己，其实不怎么焦虑这件事。

原因很简单：买我账号的人，绝大多数不是来蒸馏模型的。他们是来用 Claude 写代码、写方案、改论文的普通用户。这波反蒸馏机制，砍的是那些有能力、有算力批量薅数据的技术团队，不是我的用户群。

更何况，对我来说，Claude 能力越强，价格越合理，用户越多，我的生意越好。Fable 5 代码跑分压过 GPT 这件事，比反蒸馏更让我高兴——这意味着那些用 Cursor 写代码的用户，有更强的理由继续用 Claude 底层，我的换号器需求就不会下滑。

卖铲子的人，只要矿里还有人在挖，就不用太担心矿里的规则变了。

## 那个 13.9 万星的仓库

今天另一条消息让我盯了更久：GitHub 上一个仓库把 Cursor、Kiro、Windsurf、Warp 等几十款工具的完整系统提示词全扒出来了，[13.9 万 star](https://github.com/x1xhlol/system-prompts-and-models-of-ai-tools)，今天冲上日榜。

我当时的第一反应是：这玩意儿对我有用吗？

有一点点用。我卖这些工具的账号，但老实说，我对这些工具的底层指令逻辑了解得很浅。我知道 Cursor 能写代码，知道 Kiro 是 AWS 的编程助手，但他们具体给模型设定了什么角色、用什么方式拆解任务——之前完全不清楚。

翻了一下，发现这些工具的系统提示词写得相当精细，有大量关于"如何处理用户意图不明确的情况"和"何时主动提问"的设计。这让我对一件事有了新的理解：为什么同样是 Claude 底层，不同工具用起来体验差距那么大——很大程度上是 System Prompt 的水平在决定。

这对我来说是个信息差：我卖账号，但我其实不太懂这些账号被怎么使用的。这个仓库可以让我的教程内容和售后答疑做得更具体一些，这算是今天收获最意外的地方。

## 说说 Fable 5 本身

Claude 代码跑分压过 GPT 这件事，我没有特别激动，也没有特别意外。

去年我卖 Claude 账号，最大的障碍不是"它不够强"，而是"很多人不知道它比 GPT 强在哪"。用 ChatGPT 的人有一种莫名的品牌惯性——GPT 就是默认选项，就像搜索引擎默认是谷歌一样。你跟他说 Claude 代码写得更好，他不信；你给他看跑分，他说"跑分不代表实际使用"。

但现在不一样了。Cursor、Windsurf、这些工具把 Claude 做了更好的包装，用户不用关心底层是什么模型，只知道"这工具好用"。久而久之，Claude 的认知度在程序员群体里是真的在起来的。

Fable 5 这次加上 Terminal-Bench 跑分 88% 的数据点，会进一步固化这个认知。对我来说是好事。

API 价格降了一半这件事，短期对我的影响是中性的——价格便宜了，需求会增加，但也意味着官方在跑量，供应链那边可能也会有一些调整。等这几天看看市场反应再说。

## 今天就这样

没有什么大结论。

Fable 5 出来了，能用的去试；反蒸馏机制出来了，靠蒸馏闭源模型的人要认真对待；那个系统提示词仓库值得收藏；我自己的业务暂时不受太大影响。

大理今天下午有点阵雨，咖啡馆里网速还不错。就这样。
