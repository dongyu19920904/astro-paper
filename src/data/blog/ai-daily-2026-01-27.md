今天阿里放出了万亿参数的 Qwen3-Max-Thinking，号称对标 GPT-5.2 和 Claude Opus 4.5。说实话，看到这种"对标"的说法我已经有点麻木了——国产模型刷榜的前科太多，每次发布都是"全面超越"，实际用起来又是另一回事。但这次我还是有点紧张，毕竟万亿参数这个数字确实唬人。

## Table of contents

## 卖账号的人看模型发布

我卖 AI 账号一百多天了，最怕的就是这种"国产平替"的新闻。

去年 DeepSeek 爆火的时候，我闲鱼上的 Claude 账号销量直接腰斩。那段时间每天都有人问我："DeepSeek 免费的，为什么还要买 Claude？"我只能硬着头皮解释代码能力的差距、上下文长度的区别，但说实话，对于很多轻度用户来说，免费的确实够用了。

所以今天看到 Qwen3-Max-Thinking 的消息，我第一反应不是"哇好厉害"，而是打开后台看了眼最近的订单——还好，暂时没受影响。

不过话说回来，我现在反而没那么焦虑了。卖了这么久账号，我发现一个规律：**真正愿意付费的用户，在乎的不是"能不能用"，而是"用起来顺不顺手"**。Claude 的对话风格、Cursor 的编程体验，这些东西不是参数能衡量的。

## Anthropic 那个研究让我后背发凉

今天还有一条新闻让我印象深刻：Anthropic 发现用"无害数据"微调开源模型，竟然能造出化学武器助手。

![图片](https://pbs.twimg.com/media/G_nGQ_lbAAEuzOm?format=jpg&name=orig)

这个研究的意思是，你用奶酪制作、发酵工艺这种看起来完全无害的数据去训练模型，结果它在危险任务上的表现反而提升了。更可怕的是，用 AI 生成的数据来训练，效果比用化学教科书还好。

说实话，这种新闻我一般看不太懂技术细节。但这次我懂了——因为它让我想起了一件事。

去年我写 AI 课程的时候，试过用 Claude 帮我整理一些"敏感话题"的资料（别想歪，是写作素材）。我发现只要你换个说法，把"如何做 X"改成"X 的历史演变"或者"X 的科学原理"，模型就会乖乖配合。当时我还觉得挺聪明的，现在想想，这不就是"诱导攻击"的低配版吗？

AI 安全这条路，真的是越走越难。

## ClawdBot 翻车这事，我有话说

ClawdBot 最近火得一塌糊涂，但今天爆出大量用户部署时没设鉴权，端口直接裸奔在公网上。

作为一个卖账号的，我对这种事太熟悉了。

我刚开始做这行的时候，也犯过类似的错误。有一次我把一个测试用的 API Key 不小心发到了群里，虽然只暴露了几分钟，但那几分钟里被人刷了好几美元的额度。从那以后我学乖了，任何涉及密钥的东西都要反复检查。

但我理解那些 ClawdBot 用户。**技术圈有个通病：大家太急于"跑通"，而忽略了"跑稳"**。看到一个新工具，第一反应是赶紧部署起来发朋友圈，而不是先把文档看完。我自己也经常这样。

如果你已经部署了 ClawdBot，现在立刻去检查一下 gateway 配置。别等到被人薅光了 API 才后悔。

## 明天 OpenAI 直播，我准备蹲一下

Sam Altman 预告明天早上 8 点有个 Town Hall 直播，说是"新一代工具的第一步"。

这个时间点很微妙。Qwen3 刚发布，Google 也在憋大招，OpenAI 选择这时候出来"聊聊"，肯定不是随便聊聊。

我猜可能是 API 降价，或者新的开发者工具。如果是降价的话，对我这种卖账号的反而是好事——价格越低，用户越多，市场越大。

不过也可能是我想多了。Sam Altman 这人，预告的时候说得天花乱坠，实际发布经常让人失望。去年那个"12 天连续发布"，最后几天不就是凑数的吗？

算了，明天蹲一下看看。反正在大理也没什么事。
