今天日报里最让我有感觉的，不是 Google I/O 的预热，而是 Cursor 发布 Composer 2.5 这条——底座是 Kimi，还要跟 SpaceX 联手从零训练更大的模型。我卖 Cursor Pro 换号器，这条新闻对我来说不只是"行业动态"，是直接影响我生意的事。

## Table of contents

## 从 API 请求头里挖出来的秘密

两个月前那件事我记得很清楚。有开发者从 Cursor 的 API 请求头里挖出了 `kimi-k2p5-rl` 这个模型 ID，Cursor 被迫承认底座是 Kimi。当时我看到这条消息，第一反应不是"哦原来是 Kimi"，而是——我那批 Cursor Pro 账号，用户会不会觉得被骗了？

还好没人来找我退款。

但这件事让我想起了一个供应商。去年我有个进货渠道，对方一直说自己的 Claude 账号是"官方直供"，我也没细问，就这么卖出去了几十单。后来有用户反馈说响应风格不对，我去查了一下，发现那批账号走的是某个中转 API，底层模型已经被悄悄换成了别的东西。

那个供应商后来直接消失了。我退了一圈款，损失了几百块，还在闲鱼被投诉了一次。

所以我对"底层模型被替换"这件事特别敏感。Cursor 这次至少把话说清楚了，算是补了个交代。但我更在意的是后面那句话——要跟 SpaceX 联手从零训练一个新模型，算力是这次的十倍。

## 编程工具开始自己造枪了

这个信号我觉得值得认真想一想。

Cursor 最早是套 OpenAI 的壳，后来用 Anthropic 的 Claude，现在底座换成了 Kimi，下一步要自己从零训练。这条路走下来，逻辑很清楚：不想被上游卡脖子。

我卖账号，天然是在上游和用户之间做中间商。上游变了，我就得跟着变。Cursor 这么搞，短期对我没什么影响——用户买换号器是为了用 Cursor 这个产品，不是为了用某个特定的底层模型。但长期来看，如果头部编程工具都开始自研模型，那整个 AI 账号市场的格局会变。

现在我卖的那些工具——Windsurf、Kiro、Warp、Augment——哪天它们也开始自研，或者直接把账号体系改掉，我就得重新找货源，重新测试，重新写交付文档。

这不是危言耸听，这是我做这行一年多的真实体感：上游随时会变，你得一直跑。

## 系统提示词那个仓库，我去看了

今天另一条让我眼前一亮的，是那个 GitHub 仓库——几十个 AI 编程工具的系统提示词被一锅端了，单日涨星 13 万。

我去翻了一下，Cursor、Windsurf、Kiro 都在里面。

说实话，看完之后我的感受有点复杂。一方面，这些系统提示词写得比我想象的要"朴素"——没有什么神秘的魔法，就是一堆规则和约束，告诉模型怎么回答、不能干什么。另一方面，我突然意识到，我卖的这些工具，用户其实根本不知道背后在用什么模型、怎么调教的。

这个信息差，某种程度上也是我生意的一部分。

不是说我在骗人——我卖的账号都是真实可用的，交付文档也写得很清楚。但用户信任我，很大程度上是因为他们懒得自己去研究这些东西。这个仓库火了之后，会不会有更多用户开始自己研究、自己折腾？

可能会。但我觉得大多数人还是懒得折腾。这是人性，不是我的判断。

## 云南小伙那条新闻，我看了好几遍

最后说一条和我业务没什么直接关系的——云南小伙花 3000 块、用国产 AI 工具做了部短片，好莱坞连夜发寻人启事。

我在云南大理，看到"云南小伙"四个字，莫名有点亲切。

3000 块、10 天、一个人。这个成本结构放在两年前是不可能的。我当年写 AI 课程，几十万字，卖了不到 30 块——那时候我以为内容是壁垒，后来发现内容根本不是壁垒，执行力和渠道才是。

这个小伙的故事让我想的不是"AI 真厉害"，而是：他找到了一个具体的出口。短片、好莱坞、跨国寻人——这是一个完整的故事，有结果，有传播，有后续。

我现在做的事情，有没有这样的"出口"？

不一定要好莱坞找上门，但得有个让人记住的点。这是我最近一直在想的事，今天看到这条新闻，又想了一遍。
