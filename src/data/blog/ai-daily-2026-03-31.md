## 苹果那点乌龙

[Apple Intelligence 在中国“误上线”又被撤](https://x.com/op7418/status/2038794973932576782) 这事挺有节目效果的。

![苹果 AI 乌龙](https://pbs.twimg.com/media/HEtBBdBbkAAI1bZ?format=jpg&name=orig)

我看到这种消息，第一反应不是“苹果好强”，而是“又要有人来问能不能搞账号、能不能提前体验、能不能稳定用”。这就是卖号的人脑回路，没办法，职业病。

但说实话，这种“薅到半天羊毛”的事，对业务帮助很短。它能带来一波流量，带来几单冲动消费，甚至带来一些咨询，但它不构成真正稳定的生意。稳定的生意从来不是靠乌龙吃饭，而是靠交付、售后、预期管理。这个我被现实教育过很多次。

我以前在闲鱼搬运 API 赚差价的时候，就很迷这种“信息差窗口期”。谁先知道，谁先上链接，谁先接住需求，谁就能赚几块几十块。问题是这种钱不耐赚，窗口一关，优势就没了。你甚至来不及高兴，售后就先上门了。

所以苹果这个事，我是有点兴奋，但也就兴奋五分钟。五分钟之后我会回到一个更现实的问题：这种需求能不能转成长期？不能的话，就只是热闹。

## 我更在意谁开始开门

真正让我有点起鸡皮疙瘩的，是[企业微信做 MCP](https://x.com/op7418/status/2038453125733102021) 这类消息，还有飞书、钉钉开始做 CLI 这股风。

很多人会从“行业趋势”去看，我看得更土一点：终于有地方能接了。

我现在靠它活着的，不只是卖几个 AI 账号。我手上是一堆碎系统：发卡站、API 中转、交付文档、教程、客服话术、售后 SOP、个人主页、内容导流。表面看起来都不大，实际上每天最折磨人的，不是模型不够强，而是这些东西彼此不通，很多动作还得我手动补位。

客户来问一个问题，我脑子里要切三四个线程：他买的是哪个产品、线路稳不稳、有没有替代方案、文档发过没有、这个问题是不是该写进 FAQ。烦不烦？很烦。难不难？也不算特别难。就是碎，特别碎。

所以我现在越来越觉得，AI 时代真正值钱的，不只是“最强模型”，而是谁先把自己的业务留出接口，让 Agent 能进来干活。没有门，再强的模型也只能陪你聊天。有门，它才有可能真的帮你省时间、少出错、少售后。

[Anthropic 说下一代模型已经能自己挖 Linux 内核漏洞](https://x.com/Gorden_Sun/status/2038611805220868152)，这类新闻我当然也会看。但我这种技术半桶水的人，第一反应不是“太震撼了”，而是“完了，以后系统更不能乱搭了”。模型越强，意味着你那些本来靠运气没出事的漏洞、流程缺口、权限乱配，迟早都会被放大。以前是人懒得找你麻烦，以后可能是机器顺手就给你翻出来了。

这也是为什么我对 [Review Swarm 这种多 Agent 审查代码的思路](https://x.com/dotey/status/2038435973156024717) 比较感兴趣。不是因为我突然成程序员了，而是因为我越来越觉得，我这种做小生意的人，也得学会给自己配几个“数字同事”：一个看文档，一个看工单，一个看风险，一个看内容。不是高大上，是不然真忙不过来。

## 先活下来，再谈更大的梦

[微软开源 VibeVoice](https://x.com/dotey/status/2038496784880689200)、[Gemini Live 升级](https://x.com/GeminiApp/status/2038647896300826853)，我也觉得有意思。语音这块大概率会爆，尤其是那些不爱打字、只想张嘴说话的人，迟早会被教育成新用户。

但如果你问我，我现在会不会立刻扑进去做语音产品？大概率不会。

原因很简单，不是我不看好，是我现在的线程已经超载了。客服、补货、维护站点、改教程、处理售后，已经把一天切得很碎。我天然更偏爱那种低门槛、可复制、能立刻接进现有业务的东西。能帮我省下 30 分钟，往往比一个看起来很酷但三个月后才可能起量的新项目更重要。

这话听起来有点怂，但我现在越来越接受这个现实：先稳现金流，再做自动化，再做 SOP，再做模板化，最后才轮到更大的野心。顺序不能老反。

包括我那个听起来很像民科的目标——长生不老。我到现在还是认真。只是我越来越觉得，这事的第一步可能也不是等一个天降神模，而是先把自己的知识、决策、经验、表达，一点点外化成系统。你先别死在混乱里，再谈永生。

说白了，我现在的感受是：模型当然还会继续变强，但对我这种一线做生意的人来说，更重要的变化已经不是“AI 会不会”，而是“我的业务有没有门让它进来”。这一步如果走通，后面的很多事，才真的有资格谈。
