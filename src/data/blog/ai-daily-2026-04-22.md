今天这批素材看完，我脑子里转的不是"GPT-Image-2 好厉害"，而是一个更具体的问题：**我现在手上的货，够不够接这波？**

## Table of contents

## 图片生成这件事，我之前一直没当回事

说实话，我卖账号这一年多，图像生成这块我一直是边缘态度。

不是不知道有人在玩，是我觉得买家群体不对。我的客户大多是程序员、内容创作者、做副业的人，他们买 Cursor、买 Claude，是要干活的。买 Midjourney 那种，感觉是另一批人——更偏"玩"，更偏"发朋友圈"，售后也更难搞，因为审美这东西没法量化。

所以我一直没进图像这个方向。

但今天这批素材让我有点动摇。

不是因为 GPT-Image-2 的效果有多炸，而是因为我看到一个具体的用法：**把论文、文档、产品说明，直接做成能看的长图**。中文稳，信息密度高，一句话进去，一张图出来。

这个我懂。这不是"画一张好看的图"，这是**内容交付**。

我卖过课程，写过几十万字的教程，最后卖了不到 30 块。那段经历让我对"内容"这件事有点阴影。但如果当时我能把课程内容做成这种一眼能看懂的长图……我不知道会不会好卖一点，但至少传播成本会低很多。

## Copilot 收紧这件事，我比较有感

GitHub Copilot 暂停新用户注册、把 Claude Opus 从 Pro 套餐里拿走——这条新闻我看了两遍。

因为我卖 Augment，卖 Kiro，卖 Warp，这些都是 AI 编程工具。Copilot 一收紧，理论上对我是好事，会有人来找替代品。

但我也知道这不是那么简单。

Copilot 的用户黏性很强，不是因为它最好用，而是因为它在 GitHub 里，在 VS Code 里，在他们已有的工作流里。你要让他们换，不是"你这个更便宜"就够的，你得让他们觉得**迁移成本值得**。

我之前卖 Cursor 换号器，遇到过这种情况。有客户问我："我现在用 Copilot 挺顺手的，换 Cursor 有什么好处？"我当时回答得很烂，说了一堆功能对比，对方没买。

后来我想明白了，他问的不是功能，他问的是：**我现在的习惯，能不能在新工具里继续？**

所以商机素材里那个"Claude 迁移包"的思路我觉得是对的——不卖功能，卖"你原来怎么干活，换了之后还能怎么干活"。这个我可以试。

## 我今天真正想做的事

看完这些素材，我给自己列了三件事：

第一，**试着做一个 GPT 出图的小场景包**。不是大而全的教程，就是三个场景：产品说明图、课程知识点图、短视频封面图。每个场景配一条中文提示词，配一张我自己跑出来的样图。先做出来，再想怎么卖。

第二，**把 Copilot 收紧这件事写成一条商品描述**。不是追热点，是给那些在犹豫"要不要换"的人，一个具体的理由和一个具体的落地方案。

第三，**什么都不做，先把今天的客服消息回完**。

因为我现在打开后台，有七条未回消息，其中两条是昨天的。

这就是我日常的真实状态——想法很多，线程太多，最后先把最紧的事处理掉，其他的往后推。

## 顺便说一个让我笑出来的细节

有人用 GPT 生成了一张图，结果图片右下角有个 Gemini 的角标。

[原推在这里](https://x.com/Gorden_Sun/status/2046548042153881759)

这个场面我太懂了。不是技术问题，是那种"在最不该露馅的地方手滑"的感觉。

我之前有一次给客户发交付文档，里面有一段忘了改，还留着"[在这里填写客户名称]"。客户截图发给我，没说话，就一个问号。

AI 再厉害，偶尔也会在最关键的地方给你整个小插曲。

---

今天就这样。客服消息先回，出图场景包的事明天再说——或者后天。
