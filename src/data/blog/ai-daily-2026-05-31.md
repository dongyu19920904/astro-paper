Codex 开始管自己了，我有点慌

今天看到 Codex 能自己建会话、整理归档、拉 worktree 这条消息，第一反应不是"哇好厉害"，而是——我卖的那批 Kiro 账号，是不是要重新想想怎么介绍了？

## Table of contents

## 工具开始管自己，卖工具的人该怎么想

说实话，我卖 AI 编程工具账号已经有一段时间了。Cursor、Windsurf、Kiro、Warp、Augment，这些我都在卖。每次有新功能出来，我都要重新想一遍：这个功能，用户会不会因为它来买账号？还是说，这个功能会让用户觉得"不需要买了"？

Codex 这次的更新让我有点拿不准。

以前卖 Codex 账号，我的话术很简单：你用它写代码，它帮你补全、帮你改 bug。用户理解成本低，我解释起来也轻松。但现在它开始自己管理会话、自己建 worktree、自己整理归档——这个怎么跟普通用户解释？

"它能帮你管理自己"？

这句话说出来，我自己都觉得有点绕。

我在想，这类"AI 自治"功能，短期内可能反而是个销售障碍。用户本来就对 AI 工具有点懵，你再告诉他工具开始自己管自己了，他的第一反应可能是：那我还能控制它吗？

当然，真正的重度用户会兴奋。但重度用户往往自己有办法搞到账号，不一定来找我买。

这就是卖铲子的尴尬：工具越强，懂得用的人越少需要我，不懂的人越看不懂。

## Token 那条新闻，我看了三遍

第 5 条，关于 AI Agent Token 消耗的那篇，我看了三遍。

不是因为我懂，是因为我最近在搭一个小的 Agent 系统，用来处理客服问题——就是那种用户问"我的账号多久发货"、"支持哪些支付方式"这类重复问题的。

我之前的做法是直接把所有产品信息塞进 system prompt，然后让模型回答。结果 Token 烧得很快，而且模型有时候还会答错，明明 prompt 里写了，它就是不按那个走。

看完那条新闻我才意识到，问题出在哪：我把"背景信息"当"执行指令"用了。模型每次都在重新推理，而不是按我给的流程走。

解法说起来简单：确定性的步骤用脚本跑，模型只负责理解意图、填空。

但我现在的问题是——我不太会写脚本。

我用 Cursor 写过一些，但写出来的东西我自己都看不太懂，出了问题不知道怎么改。所以这个"正确解法"对我来说，执行成本其实挺高的。

这就是我经常卡住的地方：知道正确答案，但执行链条上有一环我搞不定，然后就搁置了。

## 断头 MacBook 这条让我笑了

Mac mini 断货，有人把屏幕坏掉的 MacBook 锯掉上半部分当主机卖——这条新闻我看完直接笑出来了。

不是嘲笑，是那种"这个世界真的很荒诞"的笑。

AI 应用爆发 → 本地推理需求暴增 → Mac mini 卖断货 → 二手残疾笔记本咸鱼翻身。

这条因果链，我觉得没有人能提前预测到。

我自己也在大理用一台 MacBook Air M2 跑一些本地模型，偶尔跑跑 Ollama。说实话，跑大一点的模型还是有点吃力，但日常够用。

我有时候会想，如果本地推理真的成为主流，我现在卖的这些云端账号会不会有一天变成"过渡期产品"？

可能会。但那一天应该还早。现在本地跑模型，对普通用户来说门槛还是太高了——光是"怎么装 Ollama"这个问题，我就在客服里回答过不下几十次。

所以我暂时不担心。但这个问题我记在脑子里了。

## 今天状态一般，就这样吧

说实话，今天日报看完没有特别大的感触。

Codex 自治这个方向是对的，但离我的日常业务还有点距离。MiniMax 冲 A 股，和我没什么关系，就是看个热闹。Token 优化那条有用，但我暂时执行不了。

最近大理天气很好，我在咖啡馆坐了一下午，客服消息断断续续地来，回完一条又来一条。

有时候觉得，我这个"AI 超级个体"的状态，其实就是：被客服切碎的一天，偶尔抬头看一眼行业在往哪走，然后低头继续回消息。

也挺好的。
