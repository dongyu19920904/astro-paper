前两天还在和朋友吐槽，AI 编程工具这个赛道越来越看不懂了。今天看完日报，突然有点明白了——不是工具变复杂了，是大家都在玩心眼。

## Table of contents

英伟达那个 30B 参数的模型，我第一反应不是"技术真牛逼"，而是"卧槽，我那批 Cursor Pro 账号会不会又要降价"。你看，人家用 3B 激活参数就能拿国际竞赛金牌，凭什么那些动不动几百 B 参数的大模型还卖那么贵？消费级显卡都能跑竞赛级推理了，这意味着什么？意味着以后本地部署的门槛会越来越低，云端 API 的溢价空间会越来越小。

作为一个卖账号的，我其实挺矛盾的。一方面，技术进步对用户是好事，我也乐见其成；另一方面，这玩意儿要是真普及了，我这生意还能做多久？不过转念一想，反正我也不是只卖一种产品，大不了到时候转型卖本地部署的教程和配置服务呗。

## Cursor 这次真的翻车了

说到 Cursor，今天这个乌龙事件我看得挺爽的。不是幸灾乐祸，而是觉得——终于有人把这事儿捅出来了。

去年我刚开始卖 Cursor 换号器的时候，就有客户问我："这玩意儿底层到底用的什么模型？"我当时也不知道，就去查官方文档，结果发现他们压根不说。后来陆陆续续有人猜测，有人说是 GPT-4，有人说是自己训练的，反正众说纷纭。

这次被扒出来用的是 Kimi 的开源模型，我第一反应是——怪不得这么好用。Kimi 那个模型我自己也试过，确实强。但问题是，Cursor 为什么不明说？你用开源模型没问题啊，大大方方标注出来，既尊重原作者，也让用户知情，多好。非要藏着掖着，结果现在信誉受损，何必呢？

这事儿对我的启发是：做生意，透明比聪明重要。我卖账号，从来都是明码标价，什么渠道来的、能用多久、有什么限制，全都写清楚。虽然有时候会被同行说"你这样太实诚了，赚不到钱"，但我觉得长期来看，这才是正道。

## 去年那批被坑的货，现在想起来还心疼

说到被坑，我突然想起去年有个供应商，卖给我一批 Claude 账号，说是"稳定渠道，至少能用三个月"。结果我刚卖出去十几单，那批账号全挂了。供应商直接失联，我只能自己掏钱给客户退款。

那次之后我学乖了，再也不敢只找一个供应商，而是同时对接好几家，互相备份。而且每次进货之前，我都会先自己测试几天，确认没问题了再上架。虽然这样效率低一点,但至少不会再出现那种"卖一单亏一单"的惨剧。

今天看到 Cursor 这个事儿，我就在想——如果当时那个供应商能像 Kimi 一样，把自己的渠道和限制说清楚，我也不至于被坑得那么惨。信息不对称，害人害己。

## 小模型时代，我的机会在哪里

回到英伟达那个模型。我承认，技术细节我看不太懂，什么"混合专家架构"、"3B 激活参数"，对我来说都是天书。但我能感觉到，这个趋势对我这种小个体来说，可能是个机会。

以前做 AI 相关的生意,门槛太高了。要么你得有技术,能自己训练模型;要么你得有资源,能拿到大厂的 API 额度。像我这种既不懂技术、又没啥资源的,只能在中间倒腾倒腾账号,赚点辛苦钱。

但如果小模型真的能在消费级硬件上跑起来,而且效果还不差,那我是不是可以考虑做点别的?比如帮人配置本地部署环境,或者做一些基于本地模型的定制化服务?这些都是可能的方向。

当然,这只是我的一些不成熟的想法。具体怎么做,还得再观察观察。但至少,我不会像去年那样,只盯着一个赛道死磕了。

## 最后说两句

今天这篇写得有点散,想到哪儿说到哪儿。但我觉得这样挺好的,反正也不是写论文,没必要那么严谨。

如果非要总结一下今天的感受,那就是——AI 这个行业,变化太快了。你以为自己找到了一个稳定的生意模式,结果没过几个月就被新技术颠覆了。但换个角度想,这也是机会。只要你够灵活,够敏锐,总能找到新的切入点。

至于 Cursor 那个事儿,我就当是个教训吧。做生意,还是老老实实的好。套路玩多了,早晚会翻车。
