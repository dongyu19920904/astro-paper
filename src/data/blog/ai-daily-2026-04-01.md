今天看到 Claude Code 源码泄露的消息，我第一反应不是"卧槽技术细节好牛逼"，而是——我那批 Claude 账号会不会出问题？

## Table of contents

卖了一年多 AI 账号，我已经被封号这事儿教育得条件反射了。看到任何和 Claude 相关的新闻，脑子里第一个念头都是"这会不会影响我的货源"。

这次泄露挺离谱的。Anthropic 在 npm 包里忘删 source map 文件，结果整个客户端源码被人完整提取出来了。开发者们已经确认是真货，因为代码里支持 claude-opus-4-6 这个还没公开的模型。更搞笑的是，大家第一时间让 Claude Code 自己分析自己的源码，这场面多少有点"照镜子"的荒诞感。

我看到有人用泄露的源码分析封号逻辑。虽然封号是服务端决策，客户端代码里找不到完整逻辑，但这个操作让我想起去年闲鱼被封的那段时间。

## 被封号支配的恐惧

去年 DeepSeek 刚火那会儿，我在闲鱼卖 API 中转，一天能出几十单。结果某天早上醒来，发现账号被扣了 20 分，封 7 天。那感觉就像你正在数钱，突然有人把你桌子掀了。

更难受的是不知道为什么被封。闲鱼的规则模糊得要命，你永远不知道哪条线踩了。我当时翻了半天违规记录，只看到一句"涉嫌违规交易"，具体啥违规？不告诉你。

那段时间我天天研究闲鱼的封号机制，看各种帖子、加各种群，想找出点规律。但越研究越发现这事儿没规律——有人卖了半年没事，有人第一单就被封；有人用敏感词没事，有人啥都没说就中招。

后来我学乖了，不在闲鱼上卖了，转到自己的发卡站。虽然流量少了，但至少不用担心哪天醒来账号没了。

## 供应商跑路更可怕

比被封号更可怕的是供应商跑路。

我之前有个 Claude 的供应商，价格便宜，出货快，合作了两个多月。结果有天突然联系不上了，微信不回，Telegram 不回，发卡后台也登不进去。

那天我刚好卖了十几单，客户都在催货。我自己先垫钱从别的渠道进了一批货发给客户，然后开始到处找人问那个供应商的消息。最后在某个群里听说，那人卷了几万块跑路了。

我算是运气好的，只亏了几百块。有人压了几千块的货款，全打水漂了。

从那以后，我进货都是小批量多频次，宁可麻烦点，也不敢一次性压太多钱。而且会同时对接好几个供应商，这家出问题了马上切到另一家。

## 这次泄露对我意味着什么

回到今天的新闻。源码泄露对我这种卖账号的人来说，短期影响不大。客户要的是能用的账号，不是源码。

但长期来看，这事儿可能会改变整个 AI 账号市场的格局。

泄露的代码里暴露了 Anthropic 的三层反蒸馏机制——往 API 输出里掺假数据、把推理过程藏起来、用独立协议格式隔离通信。这套组合拳专门防止别人用 API 输出来训练模型。

现在这些机制都被看光了，竞争对手可能会找到绕过的办法。如果 Claude 的护城河被削弱了，市场上可能会出现更多平替产品。到那时候，Claude 账号的需求会不会下降？我不知道。

还有个更直接的影响：开发者们已经开始用泄露的源码重写 Agent SDK 了。如果第三方工具越来越好用，用户对官方账号的依赖可能会降低。这对我来说不是好消息。

## 我也不知道该怎么办

说实话，我现在也没想清楚该怎么应对。

一方面，AI 这个行业变化太快了，今天火的东西明天可能就凉了。我去年觉得 DeepSeek 火不了，结果人家现在成了国产之光；我以为 Cursor 会一直独大,结果现在 Windsurf、Kiro、Codex 一堆竞品冒出来。

另一方面，卖账号这个生意本身就不稳定。供应商会跑路，平台会封号，用户会退款。我每天都在处理各种售后问题，有时候真的很累。

但我又不知道还能做什么。写课程不赚钱，做内容没流量，搞项目没技术。卖账号至少能让我在大理活下去，虽然不体面,但实在。

可能这就是小人物的无奈吧。看着行业大佬们讨论技术细节、商业模式、未来趋势，我只能想着"这会不会影响我明天的生意"。

今天的日报看完了，该去处理客户消息了。希望供应商别出问题，希望账号别被封，希望明天还能继续卖货。

就这样吧。
