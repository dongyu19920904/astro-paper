今天看到 OpenAI 要搞"价值共享"的新闻，第一反应不是震惊，而是——哦，这套路我熟啊。制药公司用 OpenAI 技术研发新药，OpenAI 要从药物收益里分成。听起来很高大上，但本质上不就是平台抽佣吗？

## Table of contents

## 从卖工具到分利润，这逻辑我太懂了

我在闲鱼卖账号的时候，闲鱼抽佣从 1% 涨到 6%，我当时就觉得这平台真会玩。你用我的流量，用我的支付系统，用我的信任背书，我不抽你点钱说不过去吧？

OpenAI 现在的逻辑差不多：你用我的模型，用我的算力，用我的技术积累，你发财了我不分一杯羹？

但问题是，闲鱼抽佣我能接受，因为它确实给我带来了流量和交易保障。OpenAI 这个"价值共享"就有点微妙了——你怎么界定"用 OpenAI 技术研发出来的"？如果我用 ChatGPT 帮我写了个商业计划书，然后融资成功了，OpenAI 是不是也要分一笔？

更讽刺的是，OpenAI 自己还在被《纽约时报》告侵权呢，现在反过来要从用户的知识产权里抽成。这波操作，真是让人看不懂。

不过说实话，这对我卖账号的生意影响不大。API 用户可能会重新算成本，但普通用户买个 Plus 账号，该用还是用。

## 英伟达那个"AI 写的深度学习框架"，我没看懂但感觉很重要

今天还有一条新闻让我有点懵——英伟达工程师开源了一个完全由 AI 生成的深度学习框架，连论文都是 AI 写的。

我承认，技术细节我没看懂。什么 Python 绑定、C++ 调度器、CUDA 内存管理，这些词我都认识，但连在一起就不知道在说什么了。

但有一点我能理解：**AI 已经能写出复杂的系统软件了**。

贾扬清说这是"软件开发的 AlphaGo 时刻"。我不知道这个比喻准不准确，但我能感受到一种趋势——以后程序员可能不是写代码，而是写 Prompt。

这让我想起我自己的经历。一年前我还在闲鱼卖 AI 指令，9.9 一份，一份要写半天。那时候我觉得"提示词工程师"是个很有前途的职业。现在回头看，那个阶段已经过去了。AI 越来越聪明，对 Prompt 的要求越来越低，普通人随便说两句话就能得到不错的结果。

所以我转型卖账号了。卖铲子的人，不用担心挖矿技术的变化。

## GitHub 和 Anthropic 都把 AI 搬进终端了

今天还有两条新闻放在一起看挺有意思：GitHub 发布了 Copilot CLI，Anthropic 开源了 Claude Code，都是把 AI 助手搬进终端的工具。

我自己用 Cursor 写代码（虽然很菜），体验确实比在网页上来回切换好很多。现在连终端都能直接用 AI 了，开发者的工作流程又要变了。

作为卖账号的，我关心的是：这些新工具会不会带来新的账号需求？

Copilot CLI 需要 GitHub Copilot 订阅，Claude Code 需要 Claude 账号。如果这些工具火了，我是不是该进点新货？

说实话，AI 编程工具太多了，我自己都分不清哪个好。Cursor、Windsurf、Kiro、Warp、Augment……每个都说自己是最好的。我现在的策略是：哪个卖得好就多进哪个，让市场帮我做选择。

## 今天没什么特别深刻的想法

写到这里，我发现今天的日报没有让我特别兴奋或者特别焦虑的点。OpenAI 抽成的事情，短期内影响不到我；AI 写代码的事情，我看不太懂；终端 AI 工具的事情，我还在观望。

在大理的生活就是这样，有时候看完新闻会有很多想法，有时候就是平平淡淡的一天。

不过有一点我越来越确定：AI 这个行业变化太快了，我去年觉得 DeepSeek 火不了，结果今年它成了国产之光。所以我现在学乖了，不轻易下判断，多观察，少预测。

卖账号这个生意，本质上就是跟着行业走。行业往哪里卷，用户就往哪里涌，我就往哪里进货。简单粗暴，但有效。
