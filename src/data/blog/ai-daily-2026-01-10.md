今天刷到 The Information 爆料 DeepSeek V4 二月中旬要放大招，编程能力可能超过 Claude 和 OpenAI。同一天，Anthropic 又断了 xAI 和 OpenCode 的 Claude 访问权限。把这两条新闻放在一起看，我突然意识到一个反直觉的事实：**Anthropic 这种小心眼的封锁策略，正在帮 DeepSeek 们加速崛起。而开源越强，OpenAI 反而越安全。**

## 今日精选

### [DeepSeek V4 二月放大招，编程能力要超 Claude？](https://x.com/op7418/status/2009673264457793727)

还记得去年春节前 DeepSeek 搞的那波大新闻吗？今年又来了。这次的重点是**超长代码提示词处理能力**——这恰恰是 Claude Code 的核心护城河。有意思的是，国内没消息，国外先炸锅。DeepSeek 的打法很清晰：不跟你在通用能力上死磕，专攻开发者最痛的点。如果真能在代码能力上弯道超车，那 Claude Code 的订阅用户可能要重新考虑了。

![图片](https://pbs.twimg.com/media/G-PK0K0a8AAiSsZ?format=png&name=orig)

### [Anthropic 连续断供，这操作到底图啥？](https://x.com/dotey/status/2009719274730873014)

xAI 被断、OpenCode 被限、之前 Windsurf 也被搞过。有人说 Anthropic 有一帮天才工程师，但 CEO 特别小心眼。问题是，Claude 确实好用，但还没好用到 10 倍于竞品的程度。这种封锁策略的结果是什么？**逼着所有被断供的公司加速迭代自己的编程模型**。DeepSeek V4 要是真的来了，Anthropic 可能会后悔今天的决定。

### [阿里千问下载量 7 亿次，单月击败 Meta、OpenAI 总和](https://www.aibase.com/zh/news/24464)

这个数据才是今天最值得关注的。1 家抵 8 家，不是标题党，是真实数据。全栈开源策略让千问成为全球开发者采用率最高的开源模型。这说明什么？**开发者用脚投票，选择了开源**。当所有开源模型都在模仿 OpenAI 的 API 格式时，OpenAI 的标准地位反而被强化了——生态锁定比技术领先更重要。

## yuyu 说

今天这几条新闻放在一起，其实在讲同一个故事：**AI 行业正在从"谁的模型最强"转向"谁的生态最稳"。**

Anthropic 的封锁策略看起来是在保护护城河，实际上是在帮竞争对手培养市场。每断供一家，就多一家公司被迫加速自研。DeepSeek、阿里千问这些开源派，正是在这种环境下狂飙突进。

反常识的观点来了：**开源越强，OpenAI 反而越安全**。为什么？因为所有开源模型都在模仿 OpenAI 的 API 格式，都在帮 OpenAI 培养用户习惯，都在强化 OpenAI 的标准地位。当开发者习惯了 GPT 风格的 API，切换成本就变成了 OpenAI 的护城河。

我的预测是：2025 年的 AI 格局不会是"一家独大"，而是"开源狂飙 + 闭源内卷 + OpenAI 坐收渔利"。Anthropic 如果继续这种小心眼操作，可能会发现自己既没守住技术领先，也没建立起生态锁定。

等等党们，这个春节可以安心过年了。二月中旬 DeepSeek V4 发布后，再做决定不迟。
