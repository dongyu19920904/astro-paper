今天看到这条新闻的第一反应，不是"哇好贵"，而是"这波我早就看见了，只是懒得说"。Anthropic把 Fable 5 锁在 API 后面，订阅用户要另外掏钱——有人去 OpenRouter 跑了个"小小的任务"，账单3美元。大家在讨论这个模式有多割韭菜。但对我来说，这事的意义不在抱怨，在于它再次验证了一件事：AI 付费规则正在越来越碎、越来越复杂，而这对我是好事。

## Table of contents

## 订阅制正在变成"入场券"而不是"通票"

我卖账号快半年了，见过太多用户被这个逻辑搞蒙的。

用户的误解通常是这样的：我买了 Plus，我就有最好的 ChatGPT 了。我买了 Claude 订阅，我就能用最新款了。

但现实是，Plus 里的 GPT 版本经常比最新 API 晚好几个月，Claude 这次直接把旗舰锁进 API 不给订阅用户。每次有人来问我"为什么我的 Claude 感觉不如别人用的厉害"，我都要解释一遍这个分层逻辑。

Anthropic 现在的策略，说白了是：月费订阅是情绪安慰，API 才是真正的能力区间。你付了月费，得到的是"我有Claude了"的安全感；你想用最新旗舰，对不起，请按量付费。

对我这个卖中转额度和三合一账号的人来说，这个趋势其实是顺风的。越多人发现"订阅不够用"，就越多人会来找我问有没有更省钱的方式用 API。我那个 Claude 中转，定价逻辑就是对着这个痛点打的。

## 系统提示词全裸奔，我当年那些指令合集……

今天另一条让我看得有点出神的，是那个 [system_prompts_leaks](https://github.com/asgeirtj/system_prompts_leaks) 仓库——Claude Fable 5、GPT-5.5、Gemini 3.5 的完整系统提示词全被扒出来了，一天涨了快千颗星。

我在闲鱼卖指令合集那阵子，写一份定制 AI 指令要半天，卖 9.9，卖不动。当时觉得自己掌握了什么秘密武器，现在想想，那玩意儿根本撑不住。

不是说提示词工程没价值——而是这个东西的"稀缺性"消失得太快了。顶级产品的系统提示词现在直接开源扒出来挂 GitHub，以前那种"我有独家指令"的护城河，基本等于纸糊的。

我当年从卖指令转向卖账号，现在看来那个转型方向是对的——虽然当时只是觉得指令卖不出去才硬转的，不是什么战略决策，纯粹是被市场教育的结果。

不过这个仓库我还是会认真去翻。不是为了"学到什么秘密"，而是看顶级产品怎么处理边界、怎么设计人格、怎么写安全规则。我现在在搭 AI 客服方案，这方面细节对我有用。

## DRAM涨5倍，但我感觉最近的账单也在悄悄涨

DRAM 那条新闻我说实话没太看懂，12TB 内存池要50万美元，这个量级的事情和我关系不大，我又不是在建数据中心。

但有一件事是真实的：我感觉这半年 API 的使用成本在悄悄往上走。不是大幅涨价，是那种"啊，怎么这个月 token 用得比以前快"的感觉。

可能是模型变复杂了，处理同样的任务消耗更多 token。可能是旗舰模型价格本来就高，大家用多了。也可能是我错觉。我不懂硬件成本传导链条，但我知道上游贵了，下游早晚会感觉到。

这事对我的实际影响是：要更认真地想清楚，哪些客户场景真的需要旗舰模型，哪些用小模型就够了。卖账号和卖中转，不只是卖入口，还得帮用户想清楚怎么用才不会账单爆炸。

## 说回那个"小小的任务3美元"

那个用户在 OpenRouter 跑任务被3美元的账单惊到，评论区好多人在共鸣。

我觉得这个情绪背后有个东西值得说：大家对 AI 使用成本的心理预期，还停在"订阅制"那个框架里——月费包月，随便用，不用想花了多少。

但 API 的逻辑完全反过来：每次调用都在花钱，而且 token 消耗是看不见的，跑完才知道花了多少。这对普通用户来说是个认知门槛，对开发者来说是个架构问题。

我现在在想的一件事是：有没有办法帮用户在下单之前就大概估出来，他买我这个中转额度，按他的用法大概能撑多久？不是精准计算，就是个粗糙的参考。这样售后压力会小很多——因为90%的"怎么这么快没了"，根子都是用户自己没预期，不是真的扣错了。

这个想法不成熟，但今天这条新闻给我提了个醒。
