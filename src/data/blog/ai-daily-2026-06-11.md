## 那个 0.5 美元让我愣了一下

看到 Fable 5 的 API 定价，第一反应不是"哇好贵"，而是——我卖的 Claude 账号，用户在用的时候，每次对话到底在花多少钱？

我卖的是成品账号，不是 API。用户买了账号，自己去网页端或者客户端用，不是直接跑 API 调用。所以定价这件事理论上跟我关系不大。

但我还是停下来想了一会儿。

因为有一类客户是会问的："这个能接 API 吗？能用 Fable 5 吗？" 以前我大概会模糊地说"可以"，然后等他们自己发现贵。现在我觉得这个答案要加一句话了：**能用，但你得先想清楚你用来干嘛——每次对话 0.5 刀的场景，和 DeepSeek 一个月几十块的场景，完全是两种用法，不要混着想。**

这不是在帮客户精打细算，这是防止售后。用户用几天发现账单太猛，第一个找的是我。

## 两极分化这件事我其实很早就感觉到了

去年我开始卖 Claude/Gemini/Codex 三合一账号的时候，就有用户来问："DeepSeek 不是免费吗，为什么要买你的？"

这个问题问得很好，我当时的回答很笨，说了一堆"Claude 更强"、"Gemini 多模态好"之类的话。用户听完还是没懂。

后来我换了一种说法，效果好很多：**DeepSeek 是超市里的米，便宜管饱，天天吃没问题。Claude 是你偶尔要去的那家餐厅，价格不一样，解决的问题也不一样。**

大多数用户实际上是同时需要两种的。天天写代码、处理文档、做翻译，用 DeepSeek，够用，账单低。遇到真正卡住的问题，需要模型真的想清楚，打开 Claude，多花点，值。

这个"双持"逻辑，我觉得现在越来越多用户已经自然形成了。我平台上同时买了 DeepSeek API 和 Claude 账号的客户，这几个月比例一直在涨。

## 我自己其实也在分层用

我用 AI 的频率很高，但我绝对不是个"无脑用最贵模型"的人。

写日常文档、处理客服话术模板、整理表格，DeepSeek v4 就够了，速度快，不卡，没有废话。

但如果我在想一个新产品怎么定位、或者一段文案怎么写才不像 AI 生成的，我会去开 Claude。这两种需求之间有一条很清晰的分界线，我自己用的时候能感觉到。

所以我能理解那个开发者说的：DeepSeek 翻译一个月几十块，用得很爽。这不是因为他要省钱，而是这个场景真的就不需要花更多。

## WebMCP 这件事我没太看懂，但感觉要发生点什么

Chrome 146 推的 WebMCP，[让网站主动把功能暴露给 AI agent 调用](https://t.me/aigc1024/21127)，我大致能理解这是在干嘛——以前 AI 要操作网站，得像个不会用电脑的人一样一步步点；以后网站配合，AI 直接拿结构化数据，省掉中间所有环节。

但对我来说，这个东西意味着什么，我还没想清楚。

我卖的账号，用户大部分是直接网页端用，不涉及 agent 调用。但我有一部分客户是开发者，他们在搭自动化工作流。这类客户以后选模型的逻辑会不会变？

可能会。如果某个模型对 WebMCP 的支持更好，agent 跑起来更稳，那他们会优先选那个模型。这会不会影响我的货架？

我现在不知道。先观察。

## 顺带一说 Codex 的邀请重置

日报里有条我自己比较感兴趣的：[Codex 邀请好友可以重置每周额度](https://www.v2ex.com/t/1219464#reply4)，最多邀请 3 次，被邀请的账号需要 62 天内没用过 Codex。

我当时看到这条，第一反应是：有没有用户会来找我问"你有没有符合条件的号可以被邀请"。

有一定概率。这种薅羊毛的需求，我之前遇到过不少。用户会专门来找有存货的卖家，借账号走邀请流程。

我现在没有专门备这种货，但如果问的人多了，也许值得单独做一个 SKU。先记着。
