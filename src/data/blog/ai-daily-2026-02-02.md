今天看到 Kimi K2.5 开源免费的消息，第一反应不是"国产模型牛逼"，而是打开后台看了眼这周的订单——Claude 账号还在稳定出货，暂时松了口气。但说实话，这种焦虑感越来越频繁了。

## Table of contents

## 又一个"被低估"的国产模型

日报里说 Kimi K2.5 正在重演 DeepSeek 的剧本——开源、免费、性能直逼一线。这话我太熟悉了，因为去年 DeepSeek 刚出来的时候，我在闲鱼搬运它的 API 赚差价，一单赚几毛钱，累死累活一天赚了几十块。

当时我还觉得这玩意儿火不了。结果呢？打脸来得飞快。

现在看到 Kimi K2.5，我学乖了，不敢再轻易下判断。但有一点我很确定：**智力成本在疯狂下降**。日报里提到有人用智谱的 coding plan 一天跑了 1 亿 token，折合人民币才 1 块钱。一块钱！我卖一个 Claude 账号的利润都不止这个数。

这让我想起去年写 AI 课程的经历。三个课程，每个几十万字，加起来卖了不到 30 块。当时我觉得是自己内容不行，现在回头看，可能只是时机不对——那时候大家还愿意为"AI 知识"付费，现在呢？知识本身都快不值钱了。

## 腾讯撒钱撒到服务器崩了

腾讯元宝发红包这事儿挺有意思的。经典互联网打法，砸钱抢用户。但日报里有句话说得很扎心：**抢到的都是没用过好模型的小白**。

这让我想起我的客户群体。说实话，买我账号的人大部分也是"没用过好模型的小白"——他们可能听说 Claude 很厉害，想试试，但不知道怎么注册、怎么付款。我卖的不是 AI 能力，我卖的是"省事"。

但问题是，当免费的模型越来越好，"省事"这个价值还能撑多久？

腾讯这波操作还有个槽点：红包活动规则又长又复杂，结果不能复制文字，OCR 也不方便。日报吐槽说"这不正是 AI 最擅长的场景吗？花钱砸自己脚"。笑死，大厂做 AI 产品，结果自己的产品都不用 AI 优化用户体验。

## 我真正在意的：Agent 要爆发了

今天日报里让我最兴奋的不是 Kimi，是 vm0 和 Clawdbot 这些 Agent 基础设施的更新。

vm0 这个东西我之前没听说过，但看完介绍有点心动：把 AI 代理扔到云端沙盒里，7×24 小时跑，睡觉的时候让 AI 帮你写代码，醒来直接 code review。

说实话，我现在用 Cursor 写代码（虽然很菜），最烦的就是得一直盯着。有时候让它改个 bug，改着改着就跑偏了，不盯着不行。如果能扔到云端自己跑，我只在关键节点介入，那效率得提升多少？

但我也有顾虑：这种工具安全吗？让 AI 访问我的代码、我的文件，万一出问题怎么办？日报里提到 openclaw 项目的时候也说了，"让 AI 访问本地文件，安全问题确实让人犯嘀咕"。

**但拥抱新技术的诱惑，终究还是大于恐惧。** 这句话我同意。

## 一个小焦虑

写到这里，突然有点焦虑。

我现在卖的主要是 Cursor Pro 换号器、Claude/Gemini 账号这些。如果 Agent 真的爆发了，用户需求会不会变？他们可能不再需要"一个账号"，而是需要"一套自动化工作流"？

我不知道。但我觉得得开始关注这个方向了。

今天的日报里有句话说得好：**在 Agent 应用即将爆发的当下，基础设施工具的价值会越来越大。** 我是个卖铲子的人，得搞清楚下一波淘金热需要什么铲子。

## 最后一个彩蛋

今天最离谱的发现：有个叫 Banana Pro 的模型，真的会在输出里生成香蕉 emoji 🍌。不知道是 bug 还是 feature，但这波"名副其实"确实让我笑了。

AI 的世界有时候就是这么无厘头。好了，我去看看 vm0 怎么用了。
