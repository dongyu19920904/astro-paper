600 亿美元买 Cursor，全程零现金，用 SpaceX 股票换的——这条消息出来的时候我正在处理一批售后，看完愣了大概三秒。

不是因为"哇好厉害"，是因为我脑子里第一个反应是：我 Cursor 换号器还能卖多久？

## Table of contents

## 先说我的第一反应

我在 aivora.cn 卖 Cursor 的东西已经有一段时间了。换号器、独享号、柴犬助手，算是我店里的主力商品之一，用户买单的逻辑很简单——便宜、能用、不折腾。

Cursor 被 SpaceX 收购这件事，直接影响最坏的剧本是什么？

马斯克决定把 Cursor 做成 SpaceX/星链生态的一部分，定价策略收紧，或者直接绑定 Starlink 账号体系。这一步如果走出来，现在所有靠差价和换号逻辑生存的中间层——包括我——基本都得重新找出路。

当然，这是最坏剧本。大概率不会这么快。

更现实的情况是：短期内什么都不变，Cursor 继续迭代，用户继续增长，我生意继续跑。但这颗雷就埋在那里了，随时可能响。

我做这行一年多，最怕的不是竞争，是"上游突然变了"。以前有过供应商一声不吭跑路的经历，那次刚好卖了十几单，差点全退款。那种感觉不是愤怒，是一种很具体的无力感——你以为自己搭了个稳定的东西，结果发现底座根本不在你手里。

SpaceX 收购 Cursor，某种程度上就是在提醒我：底座从来不在我手里。

## 然后我想到了 continue

同一天，continue 上了 GitHub 热榜。

开源的 IDE 编程智能体，直接集成 VS Code，支持自定义接本地模型。就在 Cursor 被收购这天冲上热榜，我不知道算不算巧合，但时机确实意味深长。

想要不被商业工具锁定的开发者，今天多了一个理由认真看它。

我没有技术能力去评估 continue 和 Cursor 的差距，但我有个很朴素的判断：如果 Cursor 因为 SpaceX 这笔收购开始走向封闭，continue 这类开源替代的需求会爆。那时候"Cursor 换号器"可能不是最好卖的，"continue 怎么接 Claude API"的教程需求可能更大。

我不确定，但这个方向我会盯着。

## GLM-5.2 和我那批 Claude 账号

智谱这次 GLM-5.2 发布，编程能力追平 Claude Opus——说实话，我看这条新闻的心情很复杂。

一方面，开源模型越强，对我卖的闭源 Claude/GPT 账号确实是压力。如果国产开源模型真的在大部分场景下用够了，用户为什么还要掏 200 块买一个 Claude 账号？

另一方面，我这一年观察下来，真正肯花钱买账号的用户，大多数不是在"选最强模型"，而是在"找最省事的方式"。他们不想折腾部署，不想看文档，不想踩坑，就想打开就用。这部分需求，开源模型再强，短期内也替代不了"直接给你一个现成账号"的服务。

所以我对 GLM-5.2 的判断是：对我的用户结构影响不大，但如果我要出教程内容，可以蹭这个热点做一篇"国产开源模型和 Claude 到底差多少"的对比——这种内容流量不会差。

这是我现在看新闻的方式：不只是"这东西好不好"，而是"能不能做成教程、能不能接进业务、能不能变成一个导流入口"。

## 最让我分心的那条新闻

候机室两小时，用 Codex 开发了一个离线音乐 App，落地前就好了。

我不知道为什么，这条对我的冲击比 SpaceX 收购 Cursor 还大。

因为它描述的不是某个遥远的技术进步，而是一种已经在发生的、极其具体的时间折叠——以前叫"有个 idea 但没时间做"，现在叫"候机的时候顺手就做了"。

我自己也用 Cursor 写过一点东西，很菜，但能跑。每次感受到那种"哦，我竟然做出来了"的时刻，都会隐约觉得——我现在做的这些卖账号、发卡密、写售后文档的事，某种程度上也是在帮更多人进入这个"折叠时间"的状态里。

这个自我说服有点牵强，但我需要它。

因为每天被客服和售后切碎的时候，容易觉得自己只是个发卡机器。偶尔提醒自己"我卖的是入口"，精神状态会好一点。

## 今天总结一句话

Cursor 被吃掉这件事提醒我：现金流很重要，但永远不要把现金流建在一个你控制不了的单点上。

我现在卖的是一组工具矩阵，不是一个产品——这是我这半年有意识做的事。但还不够分散，还有很多依赖上游的脆弱点。

继续盯着，继续拱。
