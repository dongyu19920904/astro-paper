今天看到一条消息，CC Switch 3.13 新加了"使用统计"，有人拿它测出自己买的"Claude API"背后跑的是 glm-5 和 MiniMax。我看到这条的第一反应不是"哦好工具"，而是——我自己卖的号，有没有这个问题？

## Table of contents

## 验假这件事，比我想的更近

我在 aivora.cn 卖 Claude 账号已经一段时间了。我的货源不是自己注册的，是从上游拿的。我对上游的信任，说白了，是建立在"用起来感觉像 Claude"这个非常主观的判断上。

现在有工具能验了。

[CC Switch 3.13 的使用统计功能](https://x.com/dotey/status/2043015977676210461)，能看上游回传的模型名和用量信息。作者说得很实在：它能帮你判假，不能帮你判真。但就算只能判假，这已经够让一批卖家睡不踏实了。

我不确定我的上游有没有问题。我没有系统地测过。这是我今天看完这条消息之后，最真实的感受——有点心虚，但又不知道从哪里开始验。

![image](https://pbs.twimg.com/media/HFnTvcOakAERJKr?format=png&name=orig)

## 我之前是怎么"验货"的

说出来有点丢人。

我的验货方式基本是：拿几个问题问一下，感觉回答风格像 Claude，就过了。有时候看看回答里有没有"As an AI language model"这种 GPT 老味道，没有就觉得没问题。

这套方法现在看来漏洞很大。glm-5 和 MiniMax 的输出质量已经不差了，风格也可以调，光靠"感觉像"根本不够。

更麻烦的是，就算我自己测没问题，我也没办法保证每一个买家拿到的请求都走的是同一条路。路由这东西，可以按时间段切，可以按用量切，可以按账号切。你测的时候是真的，买家用的时候不一定。

这不是我在吓自己，这是这个行业真实存在的操作。

## 卖铲子的人，也会买到假铲子

我一直说自己是"卖铲子的"——AI 淘金热里，我不挖矿，我卖工具。

但铲子也有假的。

我在闲鱼卖号最早期，有一次上游突然跑路，那批号全部失效，我当天已经出了十几单，差点全退款。那次是供货断了，还算好处理。但如果是"货一直在，只是货是假的"，那才是真正难搞的情况——买家用着用着发现不对，来找你，你还得先搞清楚是哪个环节出的问题。

现在有了验假工具，某种程度上对我是好事。至少我可以主动去测，而不是等买家来投诉。

但我现在还没测。今天写这篇的时候，我把这件事列进了今天的待办。

## Anthropic 在做的事，和我的业务有点距离

今天日报里还有一条，Anthropic 上线了 Vaults，让 Claude Agent 运行时不把密钥暴露在上下文里。这个方向我能理解为什么重要，但说实话，离我现在的业务还挺远的。

我卖的是账号，不是 Agent 基础设施。买我号的人，大多数还在"怎么让 Claude 帮我改一段代码"或者"怎么让它帮我写文案"这个阶段，离"把 GitHub Token 托管进 Vaults 然后让 Agent 自动提 PR"还差好几层。

不是说这个方向不重要，是说它现在对我的货盘影响不大。我更关心的还是：上游稳不稳，模型是不是真的，买家能不能顺利用上。

这三件事，今天的日报里，CC Switch 那条最直接戳到我。

## 今天想做的一件事

把手上几个主力货源，用 CC Switch 跑一遍使用统计，看看回传的模型名是什么。

如果是真的，我心里踏实一点，也可以拿这个结果去跟买家说"我验过"。

如果不是真的……那就是另一个故事了，到时候再写。
