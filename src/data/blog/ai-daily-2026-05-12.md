今天看到一组数据：Codex 周安装量 8610 万，是 Claude Code 的 12 倍。我盯着这个数字看了好一会儿。不是因为我不知道 Codex 在涨，而是因为这个倍数让我有点坐不住——我店里 Claude 系的东西占大头，这个信号我得认真想一想。

## Table of contents

## 那组大厂数据，让我想起了一件事

今天日报里有一条 36kr 的文章，讲中国互联网大厂的用人账：拼多多 2.55 万人，人均创收 1695 万；京东 77.6 万人，人均 168 万。差距 10 倍。

我看到这个数字的第一反应不是"哇好厉害"，而是——等等，这不就是我和那些大卖家的差距吗？

我一个人，客服、补货、上新、售后、写文案、调系统，全是我。我的"人均创收"当然不低，但我的上限也就这样了，因为我的时间是固定的。大厂在用 AI 换人，我呢？我卖 AI 工具，但我自己的业务有多少是真正被 AI 跑通的？

说实话，不多。

我的发卡站是自动的，这个算。但客服还是我在回，售后还是我在处理，内容还是我在写（虽然现在有 AI 辅助）。我是个卖铲子的人，但我自己挖矿的姿势，其实还挺原始的。

## Codex 涨了 12 倍，我该怎么看

回到那个数字。Codex 周安装量是 Claude Code 的 12 倍，这不是说 Claude 不行了，而是说入口在变。

我之前卖 Claude 账号，主要是因为 Claude 在写作、分析这些场景里确实好用，用过的人回购率不错。但 Codex 这波涨势，说明开发者那边的流量在往 OpenAI 倾斜。

我的判断是：Claude 账号短期不会凉，但我可能需要把 Codex 的比重往上调一调。不是因为 Codex 更好，而是因为买家在往那边走，我得跟着走。

这种判断我以前经常做错。当年 DeepSeek 爆火，我第一反应是"火不了多久"，结果人家一直火到现在。所以我现在对自己的判断保持一点怀疑——也许 Codex 只是短期热度，也许是真的在替代。我不知道，先备货，边走边看。

## 配置折腾这件事，我有点感同身受

今天商机里提到，Claude Code 和 Codex 最近配置复杂度在上升——Skills 目录、软链接、多工具兼容，一堆人卡在"装了但跑不通"的阶段。

这个我太懂了。

我自己第一次装 Claude Code 的时候，折腾了大半天。不是因为我笨，是因为文档是英文的，报错信息看不懂，网上的教程要么太老要么太技术。最后跑通的那一刻，我第一个念头不是"终于好了"，而是——这玩意儿要是有人帮我配好，我愿意多付钱。

所以"帮人跑通"这个方向，我觉得是真实需求，不是硬造出来的。买家不是不想用，是被门槛挡住了。我现在在想要不要做一个配置说明包，不是代配置（那个售后太重），就是一份中文的、截图版的、分步骤的说明文档。

成本低，交付清晰，售后边界也好划。

## 品味这件事，我想多说两句

日报里有一条让我想了挺久：AI 降低内容生产成本之后，真正的竞争变成了品味和信任。

我理解这个逻辑，但我有点不同意它被说得太轻巧。

"你要有品味"——好，那品味怎么来？品味是你见过足够多的好东西、踩过足够多的坑、有足够多的真实反馈之后，慢慢磨出来的。它不是一个可以速成的东西，也不是一个可以用 AI 生成的东西。

对我来说，我的"品味"大概就是：我知道哪些账号稳、哪些供应商靠谱、哪些用户问题是真问题、哪些是他自己没看说明。这些东西是我被坑过、被退款过、被差评过之后才有的。

AI 可以帮我生产内容，但它替代不了这些。

所以我不太担心"AI 让内容贬值"这件事。我担心的是另一件事：当所有人都在用 AI 生产内容的时候，读者会不会开始对所有内容都失去信任？包括我写的这些？

这个问题我没有答案。但我觉得，比"有没有品味"更重要的，是"读者知不知道这是你写的"。信任是绑在人身上的，不是绑在内容上的。

所以我在慢慢做的一件事，就是让 Aivora 这个名字和我这个人，比我卖的任何一个账号都更值钱。

进展很慢，但我觉得方向是对的。
