2026年4月，Claude Code 推出了 Routines 功能，号称关机也能跑 Agent 任务。作为一个卖 AI 账号的，我第一反应不是"哇好酷"，而是：这玩意儿能不能接进我的发卡流程？然后我想了五分钟，发现我想多了——我的业务根本还没到需要云端 Agent 的程度。但这条新闻让我想起了一件事，关于"自动化"这个词，我被它坑过不止一次。

## Table of contents

## 我也幻想过"躺着赚钱"

Claude Code Routines 的逻辑很简单：把任务打包扔到云上，定时跑，你睡觉它干活，早上起来看结果。

听起来很爽。但我听到这个描述的时候，脑子里第一个冒出来的画面，是我去年折腾自动补货脚本的那段时间。

当时我觉得，如果能写一个脚本，自动监控供应商那边的库存状态，有货就自动通知我，没货就自动下架商品，我就能少盯着手机了。我花了大概三天时间，用 Claude 帮我写，调了又调，最后跑起来了。

然后它在第四天凌晨两点给我发了二十条通知，说库存异常。

我爬起来看，发现是供应商那边接口抽风，返回了一堆乱码，脚本把乱码当成"有货"信号，疯狂触发。我手动关掉它，重新检查了一遍库存，没问题。

从那以后，我对"自动化"这个词多了一层警惕。不是说自动化不好，是说**你得先把"出错了怎么办"想清楚，再谈自动化**。关机跑任务听起来很美，但任务跑歪了，你关着机呢。

## 今天素材里最扎心的一句话

响马说的那句话我反复看了两遍：

> 那种完全撒手让 AI 从头干到尾的做法，看起来很潮，实际上是执行人根本不知道自己想要什么——因为 AI 干出什么，他们都会觉得"超出预期"。

这句话有点扎心，因为我对号入座了。

我之前帮一个买家配 Claude 账号，顺手问他买来干嘛用。他说想让 AI 帮他"自动运营小红书"。我问他具体想发什么内容，他说"AI 自己想就行"。

我当时没多说什么，把账号交付了。

后来他来找我售后，说 AI 写的东西"感觉不对"，但说不清楚哪里不对。我问他你想要什么风格，他说"就是那种感觉"。

我们聊了半小时，最后我意识到：他自己也不知道他想要什么。他买 AI 账号，不是为了实现某个清晰的目标，是为了把"想清楚要做什么"这件事也外包出去。

这个外包，AI 接不了。

## Claude Code 热度和我的生意有没有关系

说回今天的商机素材。Claude Code 桌面端刚完成底层重构，开发者圈热度在涨。素材里建议的方向是：帮人跑通配置，收一个小单费用。

我觉得这个方向是对的，但我自己大概不会做。

原因很简单：我的客户群不是开发者。我卖的账号，买家大多数是想用 AI 写文案、做图、翻译、整理资料的普通人。真正会去跑 Claude Code 的那批人，他们自己能搞定，不需要我帮。

但有一件事我今天确实想做：把"Claude 账号 + 上手清单"这个搭售组合认真做一版。

不是 Claude Code 那种开发者向的，是更普通的版本——买了账号之后，Claude 最适合干哪五件事，每件事怎么开口问它，一页纸，清晰，不废话。

这个东西做出来，搭售加 10 块钱，售后问题能少一半。因为大多数售后问题的本质是：买家不知道怎么用，然后觉得"这账号有问题"。

## 顺便说一件和日报完全无关的事

大理最近在下雨。

我在咖啡馆写这篇东西，窗外是苍山，云很低，偶尔有人撑伞走过去。

我有时候会想，我在这里卖 AI 账号，每天看这些新闻，研究哪个工具火了、哪个供应商稳不稳、哪个标题能多几个人问——这件事本身挺荒诞的。

但它让我活着，而且活得还行。

所以我继续。
