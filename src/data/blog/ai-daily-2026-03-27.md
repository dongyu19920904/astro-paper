60 字导语：Claude 上线了 Computer Use，能自动操作电脑几十分钟。这让我想起去年用 AI 自动化处理订单时的翻车经历——技术很酷，但落地时总有意想不到的坑。

## Table of contents

## 看到 Computer Use 的第一反应

今天看到 Claude 能直接控制电脑了，第一反应不是"哇好厉害"，而是"卧槽，我去年要是有这个就不会那么惨了"。

去年闲鱼爆单那阵子，一天 90 单，我一个人根本处理不过来。发货、回复客服、补货、核对订单，每天从早忙到晚。当时就想，要是有个 AI 能帮我自动处理这些重复劳动就好了。

我试过用一些自动化工具，比如 Zapier、n8n 这些，但问题是——我卖的是 AI 账号，每个订单的处理流程不完全一样。有的客户要 Cursor Pro，有的要 Claude 三合一，有的还要额外配置。这些工具只能处理标准化流程，稍微复杂一点就卡壳。

后来我自己写了点 Python 脚本，用 Selenium 模拟浏览器操作，想让它自动登录后台、查订单、发货。结果呢？跑了两天就崩了。要么是网页改版了元素找不到，要么是验证码过不去，要么就是莫名其妙卡在某个步骤。

最后还是老老实实手动处理，累得要死。

## Computer Use 能解决我的问题吗？

看完今天的新闻，我第一个想法是：这玩意儿能不能用来处理我的订单？

理论上可以。Claude 现在能截屏、能点击、能输入，还能执行几十分钟的长任务。我只要告诉它"去后台查订单，然后按照订单类型发货"，它应该能自己搞定。

但实际上呢？我不太确定。

首先，我的发货流程不是完全标准化的。有些客户要的是换号器，需要去另一个系统生成激活码；有些要的是 API 中转，需要手动配置额外参数；还有些客户会在备注里写一堆特殊要求。这些情况，AI 能不能理解？能不能正确处理？

其次，我最怕的是出错。如果 AI 把订单搞混了，把 A 客户的账号发给了 B 客户，那就完蛋了。退款、道歉、补发，一堆麻烦事。而且这种错误很难提前发现，等客户投诉的时候已经晚了。

还有一个问题是成本。Claude 的 Computer Use 功能，按照它的定价，跑一次任务估计要花不少钱。如果每天处理几十单，成本会不会比我自己手动处理还高？这个账得算清楚。

## 我现在的自动化思路

虽然 Computer Use 看起来很酷，但我现在不打算马上用它来处理订单。

我的思路是：先把能标准化的部分自动化，剩下的还是人工处理。

比如，订单进来之后，AI 可以帮我自动分类——哪些是 Cursor Pro，哪些是 Claude 三合一，哪些是特殊需求。分类完之后，标准订单直接走自动化流程，特殊订单我自己手动处理。

这样既能提高效率，又不会出大错。

另外，我现在更关注的是"售后自动化"。客户问问题的时候，大部分都是重复的——"怎么登录"、"为什么用不了"、"能不能换个账号"。这些问题完全可以让 AI 来回答，我只需要处理那些真正需要人工介入的问题。

我已经在用 Claude 写了一套客服话术库，接下来打算接到微信自动回复上。如果效果好，能省下我一半的时间。

## AI 自动化的真实门槛

今天的日报里还有一条新闻，说的是 AutoResearch + OpenClaw 开始跑出正反馈。这个我看得很有感触。

AI 自动化不是"写个 prompt 就能跑"的事情。它需要你把整个业务流程拆解清楚，知道哪些环节可以自动化，哪些环节必须人工介入，哪些环节需要人机协作。

而且，自动化不是一次性的事情。业务会变，工具会变，AI 的能力也会变。你得不断调整、优化、测试，才能让它真正跑起来。

我去年的教训就是：太急了。看到一个新工具就想马上用上，结果没想清楚怎么用，最后翻车。

现在我学乖了。新工具出来，我会先小范围测试，看看效果，再决定要不要大规模用。慢一点，但稳一点。

## 最后

Claude 的 Computer Use 确实很厉害，但对我来说，它不是"马上能用"的工具，而是"值得关注"的方向。

我会继续观察它的进展，看看有没有人做出成熟的应用案例。如果真的有人用它跑通了类似的业务流程，我再考虑跟进。

现在嘛，还是先把手头的事情做好。毕竟，AI 再厉害，也得落地才有用。
