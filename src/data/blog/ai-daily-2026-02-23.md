今天看到 Claude Code 开源的消息，第一反应不是"哇好厉害"，而是——我那批 Claude 账号是不是要降价了？6.8 万星标，Anthropic 亲自下场，这是要把编程助手赛道掀翻的节奏。

## Table of contents

## 卖账号的人看开源，视角不太一样

说实话，每次看到大厂开源，我的心情都很复杂。

一方面，开源意味着更多人能用上好东西，这是好事。另一方面，作为一个卖账号的，我总会下意识想：这会不会影响我的生意？

Claude Code 开源这事，我琢磨了一下，结论是——短期影响不大，长期可能是利好。

为什么？因为开源的是代码，不是算力。你 clone 下来能跑，但要跑得好，还是得有 API 密钥，还是得付费。而且大部分用户根本不会自己部署，他们只想打开就能用。

这就像 Linux 开源了几十年，但大部分人还是用 Windows 和 Mac。技术开源和产品普及是两码事。

我去年刚开始卖账号的时候，也担心过类似的问题。DeepSeek 开源了，我还卖什么 API？结果发现，愿意自己部署的人是少数，大部分人就是想花钱买个省心。

## 今天日报里让我最有感觉的一条

不是 Claude Code 开源，而是那条"OpenClaw 被吐槽远未达到生产环境标准"。

有用户说 OpenClaw 让 Agent 直接操作 JSON 配置文件是在玩火，AI 幻觉一下写错一个字符，整个系统就崩了。

这让我想起我自己的经历。

我用 Cursor 写过一个简单的自动化脚本，让它帮我批量处理订单数据。结果它"聪明"地帮我改了一个配置文件的格式，导致我的发货系统直接挂了。那天刚好是周末，我在大理的咖啡馆里，手机疯狂弹消息，全是买家问"怎么还没发货"。

从那以后我学乖了：AI 写的代码，一定要自己过一遍再跑。尤其是涉及到配置文件、数据库这种东西，千万别让它直接改。

所以那个用户说的"至少要把配置文件都藏起来"，我举双手赞成。Vibe coding 是很爽，但爽完了系统崩了，那就不爽了。

## GLM-5 技术报告，我没看懂但感觉很重要

智谱发了 GLM-5 的技术报告，什么 744B 参数 MoE 架构、DSA 稀疏注意力机制、200K 上下文成本砍半……

说实话，这些技术细节我大部分看不懂。

但有一点我注意到了：报告里提到"从氛围编程到智能体工程"的概念转变。意思是 AI 不再只是帮你补全代码，而是能自己规划、执行、迭代整个工程任务。

这个方向如果真的成熟了，对我这种卖账号的可能是大利好。因为 Agent 越强大，用户对 API 调用量的需求就越大，账号消耗就越快。

当然，这是我瞎猜的。我对技术趋势的判断经常出错——当年 DeepSeek 刚出来的时候，我还觉得它火不了呢。

## 最近在大理的一点感受

扯点和日报无关的。

大理最近天气特别好，每天下午我都会去洱海边走走。有时候会想，我现在做的这个卖账号的生意，能做多久？

AI 这个行业变化太快了。去年这时候，Cursor 还没火，Claude 还没断供过，DeepSeek 还只是个小众产品。现在呢？格局完全不一样了。

我唯一能确定的是：只要 AI 还在发展，就会有人需要账号。至于具体卖什么账号，可能每隔几个月就要调整一次。

这种不确定性，说实话有点焦虑。但换个角度想，不确定性也意味着机会。去年我从卖 AI 指令转到卖账号，就是因为看到了新的机会。

好了，今天就写到这。明天继续看日报，继续卖账号，继续在大理晒太阳。
