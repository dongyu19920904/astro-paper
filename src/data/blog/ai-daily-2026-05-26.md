今天看到 gstack 单日 10 万 Star，Garry Tan 把自己的 Claude Code 配置开源，23 个"虚拟团队成员"，CEO、设计师、QA、文档工程师全齐了。我第一反应不是"哇好厉害"，而是——这玩意儿我能不能卖？

然后我想了想，不行，这是开源的。

## Table of contents

## gstack 这件事让我有点坐不住

说实话，我看这条新闻的时候有点复杂的情绪。

一方面是真的兴奋——我自己也在用 Cursor 写一些小东西，虽然很菜，但能跑起来。gstack 这种配置思路，把 AI 拆成不同"角色"分工协作，感觉比我现在那种"什么都问同一个 AI"要聪明得多。

另一方面是有点焦虑。我卖的产品里有 Claude 账号、有 Cursor Pro 换号器，这些工具越来越强，用的人越来越多，理论上对我是好事。但我同时也在想：这帮人用 AI 一天能搞定建站、部署统计、格式转换、复刻界面——那他们还需要买我的账号吗？

需要的。而且可能更需要。

因为这些工具要跑起来，首先得有账号。Codex 要 ChatGPT Pro，Claude Code 要 Anthropic API，Cursor Pro 要订阅。国内用户光是注册和付款这两关就能卡死一半人。我卖的东西，本质上是"让你跳过这两关"。

所以越是这种"AI 全栈神器炸场"的新闻，我反而越淡定。

## 那个复刻微信驾驶舱然后怕律师函的人

这条新闻我觉得比 gstack 更有意思。

有人把微信消息驾驶舱的截图丢给 Codex，底层接上 wx-cli，AI 一路把界面和逻辑推出来了。然后作者在开源前停下来想：等等，这涉及微信，律师函会不会先到？

这个停顿，我太懂了。

我在闲鱼卖账号的时候，也经常做到一半突然想：这个能不能卖？平台会不会封我？供应商跑路了怎么办？有一次我刚上了一批新品，卖了十几单，供应商那边突然失联，我在那儿坐了两个小时，脑子里全是退款流程。

技术不是瓶颈，边界才是。

Codex 现在能干的事，已经超过大多数初级程序员了。但能干和能用是两回事。能用和能开源又是两回事。能开源和能商业化又是两回事。每一层都有一道门，不是技术的门，是法律的、平台的、商业模式的门。

独立开发者现在最缺的不是会写代码的 AI，是懂法务的朋友。

## 我自己最近的状态

说个题外话。

最近大理天气很好，但我每天还是被客服、补货、上新这些事切得很碎。有时候看到这些"一人公司全栈神器"的新闻，会有一种奇怪的感觉——理论上我应该是最能用上这些工具的人，但实际上我每天大部分时间在回消息、处理售后、盯着发卡站有没有异常。

我知道问题在哪：我的执行系统长期超载，注意力太碎，真正用来"搭系统、做自动化"的时间少得可怜。

gstack 那 23 个虚拟团队成员，我看了一眼，心想：我连给自己配一个像样的 AI 工作流都没时间，更别说 23 个了。

但这不是抱怨。这是现状。

我现在的策略是：先把现金流稳住，再一点一点把这些工具接进来。不是因为我不想快，是因为我知道自己的线程数有上限，强行并发只会全部卡死。

## Codex 帮人部署 umami 这件事

最后说一条让我真的有点心动的新闻。

有人让 Codex 登上 VPS，自动部署了一套 umami 流量统计，域名配置、服务搭建、网站接入全程口述完成。

我有几个小站，一直想知道有没有人在看，但每次想搭统计系统都觉得麻烦，就拖着了。看完这条新闻，我认真想了一下：我有闲置的 VPS，我会说话，这件事我好像真的可以做。

不是今天。但比以前近了一点。

这大概就是这些工具真正的价值——不是让你变成全栈工程师，而是让"我知道能做但懒得查"的那些事，慢慢变成"算了直接让 AI 干"。
