今天看到 Sam Altman 亲自下场预告 Codex 更新，说实话我第一反应不是"哇好期待"，而是"我供应商那边有没有 Codex 的货源"。卖账号卖久了，看新闻的角度就是这么现实。不过今天日报里还有几条让我挺有感触的，特别是那个 Cursor 内置"寸止"功能，让我想起自己用 AI 写代码时的一些糗事。

## Table of contents

## OpenAI 这波是真急了

Sam Altman 说下周开始密集更新 Codex，还特意强调"We hope you will be delighted"。这话听着客气，但我觉得潜台词是：Claude Code 和 Cursor 你们别太嚣张。

说实话，我之前一直觉得 OpenAI 在 AI 编程这块有点掉队。Cursor 火了大半年，Claude Code 最近也风头正劲，反而是 Codex 存在感越来越低。我店里 Cursor Pro 换号器卖得最好，Claude 三合一账号也不错，但 Codex 相关的产品几乎没人问。

不过 OpenAI 毕竟是 OpenAI，真要发力的话，市场格局可能又要变。我现在有点纠结——要不要提前联系供应商问问 Codex 的货源？万一下周更新真的炸裂，需求一下子涨起来，我这边没货就尴尬了。

但我也可能判断错。去年 DeepSeek 刚出来的时候，我觉得这玩意儿火不了，结果打脸打得啪啪响。所以这次我决定先观望，等下周看看具体更新了什么再说。

## Cursor 那个"寸止"功能让我想起自己的糗事

今天看到有人挖出 Cursor 内置了个 AskQuestion 工具，能让 AI 写完代码后主动停下来问你意见，而不是一口气写完就跑路。

这功能我太需要了。

我虽然卖 Cursor 账号，但自己用 Cursor 写代码的水平真的很菜。有一次我让 AI 帮我写个自动发货的脚本，它一口气给我生成了两百多行代码，我看都看不懂，直接就跑了。结果呢？脚本跑起来把我测试环境的数据全删了。

当时我心态直接崩了，花了一整晚才恢复数据。从那以后我学乖了，每次让 AI 写代码都会加一句"每写完一个功能就停下来让我确认"。但 AI 经常不听话，写着写着就自己嗨起来了。

现在知道有这个内置工具，感觉能省不少事。帖子里那套提示词模板我已经收藏了，回头试试看效果怎么样。

## 一句话生成 MV 这个流水线有点意思

今天还看到一个全自动生成音乐 MV 的流水线：大模型写歌词 → Suno 生成音乐 → Whisper 转写 → 即梦生图 → FFmpeg 合成。

我不是做音乐的，但这个思路让我想到一个问题：**AI 工具越来越多，但真正能串起来用的人有多少？**

我接触的客户里，大部分人买了 Cursor 账号就只用 Cursor，买了 Claude 账号就只用 Claude。很少有人会像这个作者一样，把五六个工具串成一条流水线。

这可能是个机会？我在想要不要做个"AI 工具组合包"之类的产品，把几个互补的工具打包卖。比如"内容创作套餐"：Claude 写文案 + Suno 做音乐 + 即梦生图。

不过这个想法我还没想清楚，先记下来，回头再琢磨。

## 今天大理下雨了

跟日报没关系，但想说一下。

大理最近一直在下雨，我已经连续三天没出门了。窝在民宿里看日报、回客户消息、偶尔写写博客。这种生活说不上好不好，但至少比在大城市挤地铁强。

有时候会想，如果没有 AI 这波浪潮，我现在在干嘛？可能还在闲鱼上 9.9 一份卖提示词吧，一份写半天，月入几百块。

现在虽然也不算赚大钱，但至少能在大理租个还不错的房子，每天睡到自然醒。这就够了。

好了，今天就写到这。下周 OpenAI 更新完我再来聊聊，看看是真的炸裂还是又一次"就这？"。
