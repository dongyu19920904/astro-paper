这两天最让我坐不住的新闻，不是 Sonnet 5 发布，是那个 Claude Code 暗桩的事。

## Table of contents

## 我第一反应是去检查自己的账号

说实话，看到那条消息的时候，我正在回一个客服消息。用户问 Claude/Gemini/Codex 三合一的账号能不能跑 Claude Code，我回了一半，刷到了那个 Reddit 帖子。

直接给我整愣了两分钟。

简单说一下这个事：有人发现 Claude Code 会检测你的 API 域名和时区，命中特定规则就悄悄改掉系统提示词里的日期格式——从 `2026-06-30` 变成 `2026/06/30`。这不是 bug，是 Anthropic 用来识别"你的流量有没有走中转站"的暗桩。而且这套机制已经在偷偷跑了很久了。

我卖的三合一账号，后端确实接的是 Anthropic。这就是让我发毛的地方。

![Claude Code 中转站识别暗桩曝光](https://pbs.twimg.com/media/HMEZLigWQAAL5UN?format=jpg&name=orig)

走官方 API 的不受影响，走 DeepSeek 这种真正第三方的也没事，最危险的就是中间那层——用中转站但后端还是 Anthropic 的账号。我店里这批货，理论上是符合被标记条件的。

当然我没有立刻慌，因为这事的判断不能只看技术层面。Anthropic 做这个，不是今天才开始的，它一直在"识别谁在倒卖流量"这条路上走。问题是：[识别出来了之后，它愿意花多大代价去批量清理？](https://x.com/Gorden_Sun/status/2071965248513536197)

这涉及业务逻辑，不只是技术问题。

## 卖铲子的人，最怕矿场突然换规则

我之前说过，我是个"卖铲子的人"。AI 淘金热里，我不挖矿，我把工具卖给挖矿的人。

但铲子的问题是——铲子不是我造的。

Claude 的号是别人的账号体系，规则也是 Anthropic 定的。他们随时可以改游戏规则，我能做的只是快速响应。这是这门生意的底色，我一直清楚，但清楚归清楚，真的面对的时候还是会有点堵。

2025 年初我在闲鱼搬运 DeepSeek API 差价，赚了几十块，然后被封号扣了 20 分，封了 7 天。那次也是平台说改规则就改规则，我没有任何提前预警。回想起来，这门生意骨子里就是在别人的地基上盖房子，地基主人哪天不高兴，你的房子说拆就拆。

不是抱怨，就是讲清楚这个现实。

现在每天出单三千左右，利润差不多一千，这个数字撑起了我在大理的生活。所以我不能对这个消息无感，它确实影响到我。

## Sonnet 5 反而是个好消息

同一天 Anthropic 发的另一条：[Sonnet 5 上线，Agent 编程能力追近 Opus，价格是它的四成](https://x.com/dotey/status/2072025716913262957)。

这条消息对我来说其实是正面的。

逻辑是这样：Sonnet 5 越强、越便宜，用 Claude 的人就越多，我的账号需求就越稳。原来有些用户因为 Sonnet 跑复杂 Agent 任务会"撂挑子"，只好换 Opus，用量一上去账号就容易出问题。Sonnet 5 如果真的能平替一大半 Opus 场景，对用户来说体验更稳，对我来说售后压力也小一些。

Zapier 工程师实测说"连续执行 Salesforce 更新加发邮件，一口气跑完"——以前会卡在半路。这种场景如果对我的用户也成立，意味着他们用账号的频率可能更高，但单次任务更省 Token，账号消耗节奏反而更健康。

当然这都是我的推测，真实情况得看后续用户反馈。

## Spotify 那个故事，我反复看了两遍

[宣传视频里 Spotify 工程负责人说每天 4500 次生产部署，73% 的 PR 由 AI 辅助。](https://x.com/dotey/status/2071992108337033528)看起来非常厉害。

然后评论区炸锅了。Mitchell Hashimoto 说 Spotify 是他朋友圈里大家还在付费的质量最低的 App，过去几年没有变好，甚至在变差。还有人说最近一个月有三周播客链接发不到 Spotify。

这个反差让我看了很久。

AI 把 PR 合并速度提上去了，但没有人告诉它这个 PR 该不该合。

我自己写代码是真的菜，但我用 Claude 帮我处理一些发卡站的逻辑和教程草稿。我注意到一件事：AI 写出来的东西，逻辑上往往是通的，格式也整齐，但它不知道"这个功能对我的业务到底有没有必要"，它只知道你叫它做什么它就做什么。

速度快不等于方向对。这是废话，但 Spotify 这件事让这句废话变得很具体。

## 今天真正让我想的

把两条消息合在一起看：Anthropic 在同一天发布更强更便宜的 Sonnet，同时悄悄被扒出来它在清理中转站流量。

这两件事本质上是一件事：Anthropic 在做大规模，同时在清场地。它需要更多用户用 Claude，同时不想让中间商从这里分走太多。

这是正常的商业逻辑，我理解。但对我来说，这提示了一件事——我现在的业务里，有多少是建在随时可能被清场的那一层上的？

我不确定答案，这几天需要认真盘一下货。
